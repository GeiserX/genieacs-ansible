Project description: Ansible Galaxy collection (`geiserx.genieacs`) for managing [GenieACS](https://genieacs.com) TR-069 ACS instances. Provides a dynamic inventory plugin that groups CPE devices by manufacturer/model/firmware/tags, and modules for device tasks, presets, and provisions.

### Learned Patterns

- **GenieACS NBI has no pagination token** — use `limit` parameter for large device lists
- **Provisions and virtual parameters are PUT as plain text** (not JSON) — content type matters
- **Presets use `_id` as the name** — they don't have a separate ID field
- **The inventory plugin must sanitize device IDs** — TR-069 IDs contain characters invalid for Ansible hostnames

## Security Notice

> **Do not commit secrets to the repository.**
> Use environment variables (`ACS_URL`, `ACS_USER`, `ACS_PASS`) or Ansible Vault for credentials.
> The `acs_password` parameter uses `no_log: true` to prevent exposure in logs.
