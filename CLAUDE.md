# CLAUDE.md — genieacs-ansible

## Overview
Ansible Galaxy collection for managing GenieACS TR-069 ACS instances. Provides a dynamic inventory plugin that discovers CPE devices, plus modules for device tasks, presets, and provisions.

## Tech Stack
- Python 3.10+
- Ansible (>= 2.14)
- No external Python dependencies (uses stdlib `urllib` only)
- GitHub Actions CI
- Published on Ansible Galaxy as `geiserx.genieacs`

## Development
```bash
# Build collection
ansible-galaxy collection build
ansible-galaxy collection install geiserx-genieacs-*.tar.gz

# Test inventory
ansible-inventory -i genieacs.yml --graph
```

## Architecture
- `plugins/inventory/genieacs.py` — Dynamic inventory plugin (groups CPEs by manufacturer, model, firmware, tags)
- `plugins/modules/genieacs_task.py` — Create device tasks (reboot, firmware push, get/set parameters)
- `plugins/modules/genieacs_preset.py` — CRUD for presets
- `plugins/modules/genieacs_provision.py` — CRUD for provision scripts
- `plugins/module_utils/genieacs_client.py` — Shared HTTP client for GenieACS NBI API (all API calls go through here)
- `galaxy.yml` — Ansible Galaxy manifest (version, build config)
- `meta/runtime.yml` — Ansible version requirements

## Key Rules
- Never hardcode ACS credentials; use inventory file variables or Ansible vault
- Inventory files must end in `genieacs.yml` or `genieacs.yaml` for plugin recognition
- No external Python packages required beyond Ansible itself
- Part of the broader GenieACS ecosystem (container, MCP, HA, sim)
- Follow [Conventional Commits](https://conventionalcommits.org) format

## Code Style
- **Python:** Follow PEP 8, type hints where practical, `from __future__ import annotations`
- **Modules:** Include `DOCUMENTATION`, `EXAMPLES`, `RETURN` docstrings in Ansible format
- **Client:** All API calls go through `plugins/module_utils/genieacs_client.py`
- **Error handling:** Raise `GenieACSError` in the client, catch in modules and call `module.fail_json()`
- **Commit messages:** `type: description` format (e.g., `feat: add virtual_parameter module`)

## Learned Patterns
- **GenieACS NBI has no pagination token** — use `limit` parameter for large device lists
- **Provisions and virtual parameters are PUT as plain text** (not JSON) — content type matters
- **Presets use `_id` as the name** — they don't have a separate ID field
- **The inventory plugin must sanitize device IDs** — TR-069 IDs contain characters invalid for Ansible hostnames

## Security
- Use environment variables (`ACS_URL`, `ACS_USER`, `ACS_PASS`) or Ansible Vault for credentials
- The `acs_password` parameter uses `no_log: true` to prevent exposure in logs
